# Event-Driven Traffic Congestion Forecasting System
**Flipkart Grid Hackathon — Round 2**

Forecasts the impact severity of traffic events in Bengaluru and recommends manpower, barricades, and diversions — before they're needed.

---

## Quick Start (Google Colab — recommended)

1. Open **[Traffic_Congestion_Forecasting.ipynb](Traffic_Congestion_Forecasting.ipynb)** in Google Colab.
2. Run all cells top-to-bottom (**Runtime → Run all**).
3. When prompted, upload `Astram event data_anonymized - Astram event data_anonymizedb40ac87.csv`.
4. (Optional) Paste an ngrok token to launch the Streamlit dashboard.

**No GPU required. Free Colab CPU tier is sufficient.**

---

## What it does

| Component | Description |
|---|---|
| **Severity Classifier** | XGBoost model predicts `High`/`Low` priority at report time |
| **Recommendation Engine** | Rule-based: manpower count, barricade count/placement, diversion |
| **Simulated Live Feed** | Replays test-window events chronologically (clearly labelled simulated) |
| **Streamlit Dashboard** | 4-tab demo: Forecast, Hotspot Map, Live Feed, Model Report |
| **Feedback Loop (schema)** | `feedback_log.csv` captures predictions for future retraining |

---

## Scope (honest)

| Sub-problem | Status |
|---|---|
| Forecasting (severity classification) | **In scope — fully built** |
| Recommendation (manpower/barricades/diversion) | **In scope — rule-based only** (no deployment ground truth in data) |
| Real-time adjustment | **Simulated** (no live feed in static CSV) |
| Post-event learning | **Schema only** (no predicted-vs-actual pairs yet) |

## Tuning

All rule-engine thresholds (manpower bonuses, barricade counts, etc.) live in `src/config.py` under `RULES`. Change values there — no other file needs editing.

---

## Running locally (not Colab)

```bash
pip install -r requirements.txt

python -m src.preprocessing
python -m src.model
streamlit run app/dashboard.py
```
